# Prompt: Web Dashboard - Real Data Integration

**Context:**
The `Overview` and `Bell Management` pages currently use hardcoded statistics and placeholders. We need to fetch real data from the Supabase database to make the dashboard useful.

**Task:**
1.  **Overview Page (`web-dashboard/src/pages/Overview.tsx`):**
    *   Replace hardcoded numbers (Active Bells, Online Devices, Total Profiles, Storage Used) with real queries.
    *   Use `supabase.from('...').select('*', { count: 'exact', head: true })` or similar efficient queries to get counts.
    *   Calculate "Storage Used" by summing the size of files in the `audio_files` table if possible, or just count the files for now.

2.  **Bell Management (`web-dashboard/src/pages/BellManagement.tsx`):**
    *   Ensure the table fetches the actual list of devices from the `devices` table.
    *   Display real status (Online/Offline) if a `last_seen` or `status` column exists. If not, consider adding a check or assuming offline if no recent activity.
    *   Implement the UI actions (Reboot, Config):
        *   These should insert a command record into a `device_commands` table (or similar) that the ESP32 will poll.
        *   If the table doesn't exist, note this for the backend task or create a basic implementation.
